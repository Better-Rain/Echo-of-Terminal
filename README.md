# Echo Terminal

纯前端网页解密游戏原型。玩家将进入一个只读档案终端，在文件管理器、档案记录、通信镜像和辅助工具之间交叉检索，逐步还原隐藏在异常记录里的事件。

当前剧情方向以洛林公国文化部学者里希特为主角：洛林与萨克森选侯国长期战争，主角受大公密令寻找旧文明武器资料，并通过旧系统中 A 的核心席位只读镜像接触到 B、C 与战略组件相关记录。

当前仓库用于开发和发布可游玩的网页版本，内容仍处于原型阶段：美术、世界观、谜题链路和文案都可能继续调整。

## 本地运行

```powershell
npm install
npm run dev
```

开发服务器默认运行在 `http://127.0.0.1:5173/`。

## 构建

```powershell
npm run build
```

构建产物会输出到 `dist/`，可以直接部署到 GitHub Pages 或其他静态站点服务。

## GitHub Pages 发布

本项目已包含 `.github/workflows/deploy-pages.yml`。推送到 GitHub 后：

1. 在仓库 Settings -> Pages 中选择 GitHub Actions 作为发布来源。
2. 推送 `main` 分支。
3. Actions 构建成功后，页面会发布到仓库对应的 GitHub Pages 地址。

`vite.config.ts` 使用 `base: './'`，因此项目页路径不需要提前写死仓库名。

## 当前原型范围

- 类内部档案系统界面。
- 文件管理器、模拟存储介质和只读文档预览。
- 档案记录工作区，包含档案索引、正文面板和辅助软件区。
- 暗码恢复式隐藏文档：未发现前不出现在文件管理器和档案记录中，输入旧系统诊断拨号码后恢复登记。
- 辅助软件区包含通信镜像、信号接收器和本地时钟等工具。
- 数据驱动的权限规则、角色、线索标记和谜题完成状态模型。
- 示例剧情用于验证交互方向，后续可替换为正式世界观、角色、谜题和资源。

## 代码结构

- `src/domain/types.ts`: 档案、聊天、玩家身份、角色、权限规则等共享类型。
- `src/domain/access.ts`: 访问权限判断逻辑，不依赖界面层。
- `src/data/access.ts`: 角色定义和默认权限能力。
- `src/data/loginStage.ts`: 登录阶段配置。
- `src/data/fileSystem.ts`: 模拟文件系统数据，包括存储卷、目录树和文档内容。
- `src/data/player.ts`: 当前玩家身份、已发现线索和已解谜题。
- `src/data/cases.ts`: 档案数据。
- `src/data/chats.ts`: 通信镜像数据。
- `src/main.ts`: 当前渲染层和交互状态。
- `src/styles.css`: 视觉样式。

## 开发约定

- 界面文案和谜题材料尽量放在 `src/data/*`，不要直接写死在渲染函数里。
- 权限、线索和谜题完成状态优先通过 `AccessRule`、`PlayerProfile` 这类数据结构表达。
- 主界面保持系统感，避免在游戏 UI 中出现直接解释玩法的说明文字。
- 页面级滚动尽量避免，滚动应优先发生在具体面板内部。
