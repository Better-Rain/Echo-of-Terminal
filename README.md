# 黑箱档案局

纯前端网页解密游戏原型。当前目标是建立一个可以发布到 GitHub Pages 的制作环境，并提供“内部档案系统 + Unix 应用工作区”的首版美术方向。

## 本地开发

```powershell
npm install
npm run dev
```

## 构建

```powershell
npm run build
```

构建产物会输出到 `dist/`，可以直接部署到 GitHub Pages 或其他静态站点服务。

## GitHub Pages 发布

本项目已包含 `.github/workflows/deploy-pages.yml`。推送到 GitHub 后：

1. 在仓库 Settings -> Pages 中选择 GitHub Actions 作为发布来源。
2. 推送 `main` 分支。
3. Actions 构建成功后，页面会发布到仓库对应的 GitHub Pages 地址。

`vite.config.ts` 使用 `base: './'`，因此项目页路径不需要提前写死仓库名。

## 当前原型范围

- 类内部档案系统界面。
- 文件管理器、模拟本地磁盘、外接介质和只读文档预览。
- Unix 风格应用看板，包含档案索引、短波接收器、通信软件和时钟。
- 通信线索区已从固定右栏调整为独立应用。
- 示例剧情只用于验证美术和交互方向，后续可以替换为正式世界观、角色、谜题和资源。

## 当前代码结构

- `src/domain/types.ts`: 档案、聊天、玩家身份、角色、权限规则等共享类型。
- `src/domain/access.ts`: 访问权限判断逻辑，不依赖 UI。
- `src/data/access.ts`: 角色定义和默认权限能力。
- `src/data/loginStage.ts`: 第一阶段登录谜题配置，包括自检文案、账号、密保提示、负数年份和登录成功反馈。
- `src/data/fileSystem.ts`: 第二阶段模拟文件系统数据，包括本地磁盘、外接介质、目录树和文档内容。
- `src/data/player.ts`: 当前玩家身份、已发现线索和已解谜题。
- `src/data/cases.ts`: 示例档案数据。
- `src/data/chats.ts`: 示例聊天数据。
- `src/main.ts`: 当前渲染层，包含登录流程、文件管理器、Unix 应用工作区和应用切换状态。

## 访问权限模型

档案、聊天、单条消息和单个档案片段都可以挂载 `AccessRule`。规则目前支持：

- `minClearance`: 最低权限等级，例如 `L2`、`L3`。
- `anyRoles`: 任一身份满足即可，例如档案管理员或局长。
- `allRoles`: 必须同时具备的身份。
- `permissions`: 需要的权限标记，例如读取限制字段。
- `discoveredFlags`: 已发现的剧情线索。
- `solvedPuzzles`: 已解开的谜题。

这个模型是为后续谜题预留的：玩家可以通过聊天、档案推理、密码输入或伪登录等方式获得临时身份、权限标记或线索 flag，从而解锁新的档案字段。

## 第一阶段登录谜题

当前登录页流程：

1. 进入页面后播放约 2.8 秒系统自检。
2. 切换到复古终端登录页。
3. 用户名预填且不可修改，头像区域保留为待定槽位。
4. 玩家根据密保问题观察用户名编号，输入口令后进入档案系统。
5. 登录成功时显示只读权限、操作记录和欢迎信息。

当前谜题文本是可替换占位实现。正式剧情确定后，优先改 `src/data/loginStage.ts`，不要直接把谜题内容写进 UI 渲染函数。

## 第二阶段文件管理器

当前流程：

1. 登录成功后进入文件管理器工作区。
2. 文件管理器先显示短暂挂载扫描，本地磁盘会略延迟后出现。
3. 系统再延迟显示外接介质，模拟角色在登录后接入存储设备，并弹出右下角仿操作系统的可移动磁盘提示。
4. 切换存储介质或目录只会进入目录，不会自动打开文件；玩家需要主动点击文件行查看文档。
5. 本地磁盘目前包含 `SYSTEM`、`COMMUNICATIONS`、`RECORDS`、`USERS`、`TEMP` 等目录；当前登录日志和过去时间段的损坏日志集中放在 `SYSTEM/LOGS/`。
6. 玩家可以点击通知直接打开 U 盘根目录，也可以通过顶部“文件管理器”入口和左侧存储介质列表自行进入。
7. U 盘根目录包含第一份委托文档 `委托书_文化部_优先级A+.txt`，用于交代角色身份、文化部委托、只读身份和时间异常处理原则。

模拟卷、目录和文档内容集中在 `src/data/fileSystem.ts`。

## Unix 应用工作区

顶部“档案记录”入口当前进入 Unix 风格工作区：

1. 默认显示应用看板。
2. `case-index` 打开档案索引应用，沿用现有档案列表与档案正文。
3. `rx-shortwave` 打开短波接收器占位应用，后续可承载频率、音频、报码或信号图谜题。
4. `secure-comm` 打开通信软件，承载原来的安全聊天镜像。
5. `clockctl` 打开本地时钟面板，展示未校准时间、授时源状态和时间异常事件。
