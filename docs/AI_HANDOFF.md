# AI 交接说明

本文档用于新开发者或新的 AI 助手快速接手项目。内容包含实现状态和已知后续方向，会涉及当前原型流程，但不作为面向玩家的介绍文档。

## 项目概况

- 项目名称：Echo Terminal。
- 项目类型：纯前端网页解密游戏原型。
- 技术栈：Vite + TypeScript + 原生 CSS。
- 发布目标：GitHub Pages。
- 仓库地址：`git@github.com:Better-Rain/Echo-of-Terminal.git`
- 当前工作目录示例：`C:\Code\Vscode Projects\web_puzzle\Echo-of-Terminal`

## 运行方式

```powershell
npm install
npm run dev
```

开发服务器通常运行在 `http://127.0.0.1:5173/`。

构建检查：

```powershell
npm run build
```

## 发布方式

仓库包含 `.github/workflows/deploy-pages.yml`。

推送到 GitHub 后：

1. 在仓库 Settings -> Pages 中选择 GitHub Actions 作为发布来源。
2. 推送 `main` 分支。
3. 工作流会构建 `dist/` 并部署到 GitHub Pages。

`vite.config.ts` 使用 `base: './'`，因此项目页路径不需要提前写死仓库名。

5. 应用进入文件管理器工作区。
6. 文件管理器先显示挂载扫描，本地磁盘稍后出现。

12. 在 `档案记录` 左侧 QUERY 输入 `#*#*7638*#*#` 会恢复 `B_reply_record_2043.txt`、`A_reply_record_2043.txt` 和 `C_note_486479.txt`，并在左侧文档结果中显示相关文档；B 回复里出现的 `#*#*486479*#*#` 当前只会触发无效暗码提示。文件管理器查询栏也保留同一暗码入口，但不是主路径。

14. 本地磁盘包含当前登录日志和历史损坏日志，集中在 `DISK://LOCAL/SYSTEM/LOGS/`。

16. 本地 `RECORDS/signal_index_061.arc` 解析为 `RX-061 / 短波接收索引`，指向 `rx-shortwave`；`signal_log_061.arc` 在档案记录中解析为短波接收日志，给出中心频率、项目代号和 `phase_jan.arc` 相位偏移表线索。
17. 在文件管理器搜索 `phase` 可找到 `SYSTEM/INDEX/phase_jan.arc`。
18. 在档案记录中打开 `RX-PHASE-JAN / 一月载波相位偏移表`，通过表格查到 `JANUS` 对应 `1.319 rad`。
19. 顶部导航可在 `文件管理器` 和 `档案记录` 之间切换。
20. `档案记录` 打开档案工作区，右侧标签可切换通信软件、短波接收器和时钟。
21. 在短波接收器中设置中心频率 `6.107 MHz`，将相位校准调到 `1.319 rad`。
22. 接收器锁定后显示 `REPORT_NORTHLINE_061.enc` 和密钥格式。
23. 回到文件管理器搜索 `REPORT_NORTHLINE_061`，打开加密报告。
24. 用同步签名 `RX-SIG-6149` 生成校验码 `6149`，提交密钥 `JANUS-0719-6149`。
25. 报告解锁后显示北线坐标报告正文，并标记本次短波谜题完成。
26. 可根据报告正文的诊断拨号码格式 `#*#*xxxx*#*#` 推出暗码 `#*#*6104*#*#`，在档案记录侧栏 QUERY 或文件管理器查询栏执行后，系统会恢复此前不在普通目录枚举中的 `H-19_seat_residue.arc`，档案记录区同步出现 `H-19 / A席位残留索引`。

## 当前代码结构

- `src/main.ts`：状态机和界面渲染。当前阶段包括 `boot`、`login`、`authenticating`、`archive`；`activeUtilityAppId` 控制档案工作区右侧辅助软件标签；`unlockedDocumentIds` 管理本次会话内的加密文档解锁状态；`pinnedDocumentIds` 管理本次会话内的置顶文档；隐藏文档发现状态写入 `PlayerProfile.discoveredFlags`。
- `src/styles.css`：视觉样式，包括终端布局、滚动条、文件管理器、档案工作区、辅助软件标签、文档解锁表单、登录界面和 U 盘通知。
- `src/domain/types.ts`：访问规则、档案、通信、登录、虚拟文件系统等共享类型；`VirtualDocument.unlock` 用于配置加密文档，`VirtualDocument.hidden` 用于配置暗码恢复式隐藏文档。
- `src/domain/access.ts`：访问规则判定，独立于界面层。
- `src/data/loginStage.ts`：第一阶段登录谜题内容。
- `src/data/fileSystem.ts`：第二阶段虚拟存储卷、目录和文档，包括短波加密报告。
- `src/data/cases.ts`：示例档案记录。
- `src/data/chats.ts`：示例通信联系人、群组、联系人元数据、头像和消息历史。
- `src/data/player.ts`：当前玩家档案。默认是旧系统核心席位 `核心席位#0719` 的降级只读镜像；权限等级和权限列表暂不提升。
- `src/data/access.ts`：角色定义和权限能力。
- `public/assets/bureau-seal.svg`：当前本地视觉标识资产。

## 设计方向

- 风格：内部档案终端，深色背景，克制使用绿色、青色和琥珀色强调。
- 主界面避免游戏化讲解，优先使用系统式标签，例如记录、访问规则、内部备注、只读镜像、存储介质。
- 尽量避免页面级滚动；滚动应发生在具体面板内部。
- 当前滚动条为窄条终端风格。
- 界面应保持数据驱动。谜题文本优先放在 `src/data/*`，不要直接写死在渲染函数里。
- 文档类内容默认使用中文；必要的程序名、文件名、项目代号、密钥和命令名可保留原文。
