# 黑箱档案局攻略本

> 剧透警告：本文档记录当前原型版本的完整测试路线、关键线索和谜题答案。第一次游玩或做盲测时不要阅读。

本文档用于开发、测试和版本交接，不是面向玩家的无剧透介绍。

## 当前版本测试路线

1. 打开网页后等待系统自检结束。
2. 登录页出现后，用户名为只读预填账号 `中枢席位#0719`。
3. 根据密保问题输入席位编号后四位 `0719`。
4. 登录成功后进入文件管理器工作区。
5. 文件管理器会先显示挂载扫描，本地磁盘稍后出现。
6. 外接介质随后挂载，右下角出现可移动磁盘提示。
7. 点击提示中的“打开文件夹”，进入 U 盘根目录。
8. 打开 `委托书_文化部_优先级A+.txt`，确认 U 盘委托书是普通明文文件，不进入档案记录索引。
9. 等待通信提示“未投递消息已恢复”，打开沈德文会话。
10. 阅读未投递队列说明：中枢席位在原投递窗口后未再次登录，因此这条旧消息没有被原持有人读取。
11. 根据沈德文消息，回到本地磁盘，进入 `DISK://LOCAL/RECORDS/`。
12. 打开 `signal_index_061.arc`，确认文件管理器中只能看到档案源密文。
13. 切换到顶部“档案记录”，打开 `RX-061 / 短波接收索引`，阅读解析后的第二阶段入口提示。
14. 回到文件管理器，确认 `signal_log_061.arc` 是档案源文件；切换到“档案记录”，打开 `RX-061-LOG / 短波接收日志`，取得中心频率、项目代号和相位偏移表文件名线索。
15. 在文件管理器搜索框中搜索 `phase`。
16. 打开搜索结果 `DISK://LOCAL/SYSTEM/INDEX/phase_jan.arc`，确认文件管理器中只能看到档案源密文。
17. 切换到“档案记录”，打开 `RX-PHASE-JAN / 一月载波相位偏移表`，用项目代号 `JANUS` 在表格中查到相位偏移值 `1.319 rad`。
18. 切换到顶部“档案记录”，再切到右侧 `rx-shortwave`。
19. 设置中心频率为 `6.107 MHz`，相位校准为 `1.319 rad`。
20. 接收器锁定后会显示报告名称 `REPORT_NORTHLINE_061.enc` 和密钥格式 `JANUS-0719-{四位校验码}`。
21. 回到文件管理器，搜索 `REPORT_NORTHLINE_061`。
22. 打开 `DISK://LOCAL/RECORDS/REPORTS/REPORT_NORTHLINE_061.enc`。
23. 根据加密报告提示，读取接收器锁定后的同步签名 `RX-SIG-6149`，得到四位校验码 `6149`。
24. 输入完整密钥 `JANUS-0719-6149`，解锁北线镜像报告。
25. 可选：报告正文会给出新的组合词线索。回到文件管理器搜索 `灯塔二级透镜`，系统会恢复隐藏源文件 `DISK://LOCAL/RECORDS/CASES/H-19_seat_residue.arc`。
26. 切换到“档案记录”，确认新增 `H-19 / 核心席位残留索引`。

## 当前可见线索链

- 登录口令来自席位编号后四位。
- 外接介质中的 `委托书_文化部_优先级A+.txt` 是普通明文委托书，不会进入档案记录索引。
- 登录后延迟恢复的沈德文旧消息是寻找 `RECORDS/signal_index_061.arc` 的动机；未投递队列说明中枢席位原持有人没有在投递窗口后登录，因此当年未读到这条消息。
- 当前会话日志位于 `DISK://LOCAL/SYSTEM/LOGS/`，用于交代本地时钟失效、只读挂载和外接介质接入。
- `RECORDS/signal_index_061.arc` 是第二阶段入口档案源；文件管理器显示密文，档案记录解析为 `RX-061 / 短波接收索引`。
- `RECORDS/signal_log_061.arc` 在档案记录中解析为短波接收日志，提供中心频率 `6.107 MHz`、项目代号 `JANUS` 和 `phase_jan.arc` 文件名线索。
- `SYSTEM/INDEX/phase_jan.arc` 在档案记录中解析为表格，提供 `JANUS 1.319 rad`。
- 接收器锁定输出会暴露报告名和密钥格式。报告可在文件管理器中搜索，并通过完整密钥解锁。
- 加密报告的四位校验码来自锁定后的同步签名：`RX-SIG-6149` -> `6149`。
- 隐藏文档不会出现在文件管理器目录、普通搜索结果或档案记录索引中；当玩家搜索命中特定关键词后，文档恢复登记，并同时在文件管理器与档案记录中可见。
- 当前隐藏文档示例：搜索 `灯塔二级透镜`、`二级透镜` 或 `07:19 重复登录`，恢复 `H-19_seat_residue.arc` 与 `H-19 / 核心席位残留索引`。

## 关键答案

- 第一阶段登录口令：`0719`
- 第二阶段中心频率：`6.107 MHz`
- 第二阶段项目代号：`JANUS`
- 第二阶段相位校准：`1.319 rad`
- 第二阶段同步签名：`RX-SIG-6149`
- 接收器锁定报告名：`REPORT_NORTHLINE_061.enc`
- 当前密钥格式：`JANUS-0719-{四位校验码}`
- 第二阶段完整密钥：`JANUS-0719-6149`
- 当前隐藏文档发现关键词：`灯塔二级透镜` / `二级透镜` / `07:19 重复登录`

## 已知未完成节点

- 解锁状态目前只在本次页面会话中有效，刷新后不会保留。
- `REPORT_NORTHLINE_061.enc` 解锁后会写入 `PlayerProfile.discoveredFlags` 和 `PlayerProfile.solvedPuzzles`，但暂未驱动新的档案权限。
- 隐藏文档发现状态目前也只写入本次页面会话的 `PlayerProfile.discoveredFlags`，刷新后会重置。
- `clockctl` 仍是占位工具，可用于后续时间校准或纪年换算谜题。
- 核心席位原持有人姓名和头像仍未确定，但当前已明确该账号属于旧系统核心席位之一。
- 玩家进度目前没有持久化。

## 后续设计建议

- 将短波报告解锁结果接入后续权限：例如开放 B-04 的部分遮蔽字段，或解锁档案管理员临时授权线索。
- 将短波目标频率、相位校准和锁定输出移入 `src/data/*`，避免把谜题材料继续写在 `src/main.ts`。
- 给 `clockctl` 增加时间换算功能，用于后续解锁条件。
- 增加轻量持久化，记录已打开文件、已发现线索和已解锁文档。
