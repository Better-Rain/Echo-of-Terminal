# 黑箱档案局攻略本

> 剧透警告：本文档记录当前原型版本的完整测试路线、关键线索和谜题答案。第一次游玩或做盲测时不要阅读。

本文档用于开发、测试和版本交接，不是面向玩家的无剧透介绍。

## 当前版本测试路线

1. 打开网页后等待系统自检结束。
2. 登录页出现后，用户名为只读预填账号 `访客#0719`。
3. 根据密保问题输入口令 `0719`。
4. 登录成功后进入文件管理器工作区。
5. 文件管理器会先显示挂载扫描，本地磁盘稍后出现。
6. 外接介质随后挂载，右下角出现可移动磁盘提示。
7. 点击提示中的“打开文件夹”，进入 U 盘根目录。
8. 打开 `委托书_文化部_优先级A+.txt`，阅读文化部委托、只读规则、时间异常处理原则和原账号主人未核准的信息。
9. 回到本地磁盘，进入 `DISK://LOCAL/RECORDS/`。
10. 打开 `readme.txt`，确认后续内容需要使用 `rx-shortwave` 接收器读取。
11. 打开 `signal_log_061.txt`，取得中心频率、项目代号和相位表文件名线索。
12. 在文件管理器搜索框中搜索 `phase`。
13. 打开搜索结果 `DISK://LOCAL/SYSTEM/INDEX/phase_jan.txt`。
14. 用项目代号 `JANUS` 查到相位校准值 `1.319 rad`。
15. 切换到顶部“档案记录”，再切到右侧 `rx-shortwave`。
16. 设置中心频率为 `6.107 MHz`，相位校准为 `1.319 rad`。
17. 接收器锁定后会显示报告名称 `REPORT_NORTHLINE_061.enc` 和密钥格式 `JANUS-0719-{四位校验码}`。
18. 回到文件管理器，搜索 `REPORT_NORTHLINE_061`。
19. 打开 `DISK://LOCAL/RECORDS/REPORTS/REPORT_NORTHLINE_061.enc`。
20. 根据加密报告提示，读取接收器锁定后的同步签名 `RX-SIG-6149`，得到四位校验码 `6149`。
21. 输入完整密钥 `JANUS-0719-6149`，解锁北线镜像报告。

## 当前可见线索链

- 登录口令来自账号编号后四位。
- 外接介质中的委托书说明：玩家当前只拥有只读整理身份，不应修改旧文明记录或校正异常日期。
- 当前会话日志位于 `DISK://LOCAL/SYSTEM/LOGS/`，用于交代本地时钟失效、只读挂载和外接介质接入。
- `RECORDS/readme.txt` 指向短波接收器。
- `RECORDS/signal_log_061.txt` 提供中心频率 `6.107 MHz`、项目代号 `JANUS` 和 `phase_jan.txt` 文件名线索。
- `SYSTEM/INDEX/phase_jan.txt` 提供 `JANUS 1.319 rad`。
- 接收器锁定输出会暴露报告名和密钥格式。报告可在文件管理器中搜索，并通过完整密钥解锁。
- 加密报告的四位校验码来自锁定后的同步签名：`RX-SIG-6149` -> `6149`。

## 关键答案

- 第一阶段登录口令：`0719`
- 第二阶段中心频率：`6.107 MHz`
- 第二阶段项目代号：`JANUS`
- 第二阶段相位校准：`1.319 rad`
- 第二阶段同步签名：`RX-SIG-6149`
- 接收器锁定报告名：`REPORT_NORTHLINE_061.enc`
- 当前密钥格式：`JANUS-0719-{四位校验码}`
- 第二阶段完整密钥：`JANUS-0719-6149`

## 已知未完成节点

- 解锁状态目前只在本次页面会话中有效，刷新后不会保留。
- `REPORT_NORTHLINE_061.enc` 解锁后会写入 `PlayerProfile.discoveredFlags` 和 `PlayerProfile.solvedPuzzles`，但暂未驱动新的档案权限。
- `clockctl` 仍是占位工具，可用于后续时间校准或纪年换算谜题。
- 原账号主人身份和头像仍未确定。
- 玩家进度目前没有持久化。

## 后续设计建议

- 将短波报告解锁结果接入后续权限：例如开放 B-04 的部分遮蔽字段，或解锁档案管理员临时授权线索。
- 将短波目标频率、相位校准和锁定输出移入 `src/data/*`，避免把谜题材料继续写在 `src/main.ts`。
- 给 `clockctl` 增加时间换算功能，用于后续解锁条件。
- 增加轻量持久化，记录已打开文件、已发现线索和已解锁文档。
